# Pat Pat Party - PIXI 摸头动画优化

## 背景

在 0.0.3 的自定义文案基础上，继续优化摸头动画表现。目标是不改动 Token HUD、权限、冷却、GM 设置、三种力度、头顶位置校准、socket 同步与聊天卡片，只替换动画核心。

## 更新内容

- `playPatAnimation` 改为优先使用 PIXI/Canvas 临时容器播放动画。
- 动画锚点使用 Token 顶部中心 world 坐标，并继续叠加 `patOffset` 校准偏移。
- 小手从右上滑入，在 Token 头顶左右揉搓并轻微旋转，结束时上浮淡出。
- Token 视觉层增加压缩、回弹或晃动反馈，动画结束后恢复原始 mesh 状态。
- 爱心、小花、星星粒子分批冒出，具备随机大小、漂移、缩放、旋转和淡出。
- 大力逆毛摸增加短弧线和更多星星表现乱毛感，但保持软萌互动，不做攻击特效。
- 所有 PIXI ticker、临时容器和 Token mesh 反馈都会在动画结束或异常时清理。
- 保留 DOM overlay 作为 PIXI 不可用时的降级路径。

## 验证重点

- 三种力度的持续时间、揉搓次数、幅度和粒子数量应有明显差异。
- 自定义文案同时进入浮动文字和聊天卡片。
- socket 接收端也应根据 payload 的力度、文案和偏移播放同样的 PIXI 动画。
- 动画结束后不残留 PIXI 容器、ticker 回调或 Token mesh 变形。
