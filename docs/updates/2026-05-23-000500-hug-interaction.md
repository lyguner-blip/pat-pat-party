# 0.0.5 Hug Interaction Notes

## Summary

- Added a basic Hug interaction alongside Pat Pat.
- The Hug HUD button resolves a separate source token from the current controlled owned token or the user's assigned character token.
- Hug uses existing permission, cooldown, chat, and socket patterns.
- Hug animation prefers PIXI/Canvas and falls back to the DOM overlay.

## Runtime Expectations

- Select an owned token, open another token's HUD, then click the heart button.
- The dialog can accept a short custom message.
- The animation plays between the two token centers and broadcasts to other clients in the same scene.
- Chat output is created once by the initiating client when chat messages are enabled.

## Notes

- No Sequencer or third-party animation dependency is required.
- The source token must be different from the target token.
- This is a foundation for future interaction actions such as high five, shoulder pat, tea, and flowers.
